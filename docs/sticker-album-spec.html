World Cup Simulator — Feature Specification

The Sticker Album

Status Approved v1.1
Date 2026-06-29
Verified 2026-07-02 (counts still 39/12/2)
Next step Implementation
1 — Concept

A persistent meta-layer inspired by real Panini football sticker albums. Players earn stickers for elite players they draft across runs, building a collection over time. The album lives alongside the existing game - accessible at any time - and grows with every session.

Only a curated subset of elite players are collectible: roughly 53 across the current dataset (1998-2022), growing as new tournaments are added. This keeps the collection prestigious and completable.

2 — Collectible Tiers

Every collectible player belongs to exactly one tier, determined by their elo rating. Thresholds are defined in a single config object so they can be tuned without touching game logic.

// src/config.ts — add alongside FEATURES export const STICKER_TIERS = { legendary: { min: 90, max: 92 }, iconic: { min: 93, max: 96 }, monumental: { min: 97, max: 99 }, } as const // Trade-in costs: ANY duplicates (any tier/mix) needed to receive one sticker of that tier export const STICKER_TRADE_COST = { legendary: 10, // any 10 duplicates -> 1 Legendary iconic: 20, // any 20 duplicates -> 1 Iconic monumental: 50, // any 50 duplicates -> 1 Monumental } as const
Tier ELO range Current count Tournaments
Legendary 90 – 92 39 1998 – 2022
Iconic 93 – 96 12 1998 – 2022
Monumental 97+ 2 1998 – 2022
Total collectible (current dataset) 53 grows as tournaments are added
Note: 1998 and 2002 squads are partial placeholders in the current dataset. Ratings for those years should be verified before those tournaments contribute collectible players.
3 — Earning Stickers

Two triggers add stickers to the player's album. Both are gated behind the FEATURES.stickerAlbum flag (to be added to config.ts).

Timing: Stickers are never awarded mid-run. They are applied only when the run ends - either by winning the cup or being knocked out. This prevents players from restarting runs after a lucky draft roll to farm stickers without completing a tournament.
Trigger A — Run ends (win or loss)
Drafted a collectible player

Any collectible player drafted during the run is added to the album at run-end. If already owned, the duplicate counter increments. This applies whether the run ends in victory or elimination.

Trigger B — Win only
Winning the World Cup

On cup victory, after the match result screen, the player may pick any one uncollected sticker from any tier. This is a "pick your prize" moment, not a random drop. No bonus is awarded for runner-up or third place.

Playing against an opponent who fields a collectible player does not earn a sticker. The collection reflects players you personally drafted.

4 — Duplicates & Trading

Drafting a sticker you already own adds a duplicate. Duplicates are tracked per player and can be traded in for new stickers via a mini-draft UX.

Trading is cross-tier: any duplicate stickers count toward the total, regardless of their individual tier. The tier you are trading for determines the cost.

Target tier Any duplicates required Receive
Legendary 10 (any mix) 1 Legendary sticker (your choice from 3)
Iconic 20 (any mix) 1 Iconic sticker (your choice from 3)
Monumental 50 (any mix) 1 Monumental sticker (your choice from 3)

Trade UX: When the player has enough total duplicates to afford a tier, a "Trade duplicates" action appears in the album for that tier. Tapping it reveals 3 randomly drawn uncollected stickers of the target tier. The player picks one. The chosen sticker is added to the album; the duplicates are consumed from the total pool (cheapest duplicates first, or simply by count).

The 3 options are always uncollected - no risk of receiving a duplicate from a trade.

Tuning: The STICKER_TRADE_COST thresholds are a first guess. To calibrate them, the app logs a lightweight stats object to localStorage (wcsim_album_stats_v1): runsPlayed, stickersEarned, tradesCompleted. Inspect via the browser console after a few dozen runs to judge whether the costs feel right.
5 — Album Screen
  • 5.1A persistent "Album" entry point is always visible in the app (tab or button), regardless of game phase.
  • 5.2The album displays all collectible players across all tiers. Uncollected stickers appear as empty silhouette slots (name and nation visible, player image/detail obscured). Collected stickers show full detail.
  • 5.3A completion counter is always shown: e.g. "31 / 53 collected". This is the primary motivation hook.
  • 5.4Stickers are grouped by tier (Monumental, Iconic, Legendary) within the album layout.
  • 5.5When a sticker has duplicates, the duplicate count is shown on the sticker card (e.g. "x3").
  • 5.6The "Trade duplicates" action is accessible from within the album screen when the player has enough duplicates of any tier.
  • 5.7When collecting 1 or more new stickers during a run (via drafting or cup win), a post-run summary panel shows "New stickers added" with the new cards highlighted before returning to the album.
  • 5.8A 100% completion state is recognized with a distinct celebration moment. (Exact treatment: TBD in design phase.)
6 — Persistence

The album state is stored in localStorage under a versioned key (e.g. wcsim_album_v1). The schema stores collected player IDs and duplicate counts per ID. No backend required for the initial release.

  • Structure: { collected: string[], duplicates: Record<string, number> } where keys are player id fields from squads.ts.
  • Schema version: Include a version field to enable future migrations without data loss.
  • Clearing: Clearing localStorage loses all album progress. No explicit "reset album" UI required in v1, but clearing the game state should not automatically wipe the album (separate key).
  • Backend path: The localStorage schema is designed to be trivially synced to a backend later - flat object, no relational joins required.
7 — Functional Requirements Summary
  • FR-1A player is collectible if and only if their elo falls within a tier defined in STICKER_TIERS.
  • FR-2Stickers earned during a run (via drafting) are held in a pending state and applied to the album only when the run ends (win or knockout). Pending stickers do not affect album state mid-run.
  • FR-3The cup-win reward screen (shown after the match result screen) allows picking any one uncollected sticker from any tier. If all stickers are already collected, the reward is a duplicate of the player's choice instead.
  • FR-4Trading consumes STICKER_TRADE_COST[targetTier] duplicates from the total duplicate pool (any mix of tiers) and returns one sticker selected from 3 uncollected options of the target tier.
  • FR-5If fewer than 3 uncollected stickers remain in a tier, the trade UI shows however many remain (1 or 2 options). If none remain, that tier's trade option is hidden.
  • FR-6All album logic is gated behind FEATURES.stickerAlbum. When false, no album UI surfaces and no stickers are earned.
  • FR-7Album state is persisted in localStorage independently of game phase state. Resetting a run does not affect the album.
  • FR-8The post-run "new stickers" summary only appears if at least one new (non-duplicate) sticker was earned in that run.
8 — Out of Scope (v1)
  • Multiplayer sticker trading
  • Backend sync / cloud save
  • Player images or illustrated sticker art (sticker cards use text/flag only)
  • Animated sticker reveal / peel effect (nice-to-have, deferred)
  • Per-tournament or per-nation sub-albums
  • Statistics (e.g. "how many runs to collect X")
9 — Resolved Decisions
  • D-1Cup-win reward tier: Player may pick from any tier (Legendary, Iconic, or Monumental). Not restricted to the tier of the opponent faced in the final.
  • D-2Post-run sticker summary timing: Shown after the final match result screen. The flow is: match result → sticker summary (if any new stickers earned) → return to home / album.
  • D-3Album entry point: Home screen element (v1). A persistent button or section on the home/setup screen. Not a floating button or top-level nav tab.
  • D-41998/2002 datasets: Included in the collectible pool from launch. Full verified datasets for those years will be added soon; ratings will be updated then if needed.
  • D-5Trade cost calibration: A lightweight stats object (wcsim_album_stats_v1) is written to localStorage tracking runsPlayed, stickersEarned, and tradesCompleted. Inspect via browser console after ~30 runs; adjust STICKER_TRADE_COST in config.ts if the grind feels wrong.
